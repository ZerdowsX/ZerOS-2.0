# ZerOS / ZerDOS (MVP+)

Минимальная консольная ОС на **Assembly + C** с расширенной shell-системой.

## Что умеет сейчас

- загрузка через Multiboot (NASM);
- ядро на C в VGA text mode;
- цвет по умолчанию: **белый текст на зелёном фоне**;
- при старте выводится ASCII-текст `ZerDOS`;
- сборка в бинарник и ISO;
- расширенный набор команд.

## Команды

- `help` — список команд;
- `clear` — очистка экрана;
- `hello` — тестовая команда;
- `time` / `vrem` — время RTC;
- `sysinfo` — информация о системе;
- `pcinfo` — базовая информация о ПК (CPU vendor + RAM);
- `mem` — отчёт о памяти (по Multiboot);
- `fs ls | fs cat <name> | fs touch <name> | fs write <name> <text> | fs rm <name>` — файловая система;
- `fs save | fs load` — сохранить/загрузить снимок ФС на диск (`disk0`, LBA 2049);
- `ls`, `cat`, `touch`, `write`, `rm`, `save`, `load` — короткие алиасы;
- авто-установщик при старте: если ZerDOS не найден на диске, спрашивает `Install it to disk? (y/n)`;
- после установки изменения в ФС сохраняются автоматически;
- если удалить системные файлы (`system.cfg` или `boot.log`) — ОС уходит в recovery-режим и восстанавливает их;
- `color <fg> <bg>` — изменить цвет текста/фона консоли (0..15);
- `zeron edit|run|compile` — мини IDE/раннер/компиляция для Zeron-скриптов;
- `zeron help` — список тегов языка Zeron;
- сверху отображается статус-бар с режимом системы, датой и временем;
- `install` — реальная запись install-маркера через ATA PIO на выбранный диск (`disk0`, `disk1`, `disk2`) с подтверждением `YES` и проверкой чтением;
- `recovery` — восстановление важных системных файлов;
- `echo <text>` — вывести текст;
- `tester` — список тестеров (Yarida, Vlad);
- `about` — информация о проекте;
- `panic` — показать «серый экран смерти» (красный текст на сером фоне).

## Сборка

Требования:
- `nasm`
- `gcc` с поддержкой `-m32`
- `ld`
- `grub-mkrescue` **или** `grub2-mkrescue` (для ISO)
- `xorriso` (обычно нужен для `grub-mkrescue`)
- (опционально) `qemu-system-i386`

```bash
make
```

## Сборка ISO

```bash
make iso
```

## Важно про `install`

Команда `install` теперь выполняет **реальную запись** в сектор `LBA 2048` выбранного диска и делает readback-проверку.
Снимок файловой системы сохраняется в `LBA 2049` (команды `save/load`, также после успешного `install`).
Используйте только в VM/эмуляторе (QEMU/VirtualBox), если не хотите менять данные на реальном диске.

## Zeron (теги)

Сейчас поддерживаются теги:
- `[PRINT:text]` / `[ECHO:text]`
- `[CLEAR]`
- `[TIME]`
- `[PCINFO]`
- `[COLOR:fg,bg]`
- `[TOUCH:name]`
- `[CAT:name]`
- `[WRITE:name,text]`

## Запуск

```bash
make run

make run-iso
```
